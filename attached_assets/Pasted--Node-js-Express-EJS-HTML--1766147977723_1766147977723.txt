Создай веб-приложение на Node.js с Express с использованием EJS для рендеринга HTML-шаблонов. Приложение должно включать:

1. Систему авторизации:
   - Форма входа с полями "логин" и "пароль"
   - Хранение учетных данных пользователей во внешней базе данных (пока используй переменные окружения для конфигурации БД)
   - Шифрование паролей с использованием алгоритма MD5
   - Сессии для поддержания состояния входа

2. Основной интерфейс после авторизации:
   - Интерактивная HTML-таблица для отображения записей из базы данных
   - Полный набор CRUD-операций:
     * Кнопка "Добавить" с модальной формой для создания новых записей
     * Кнопка "Редактировать" в каждой строке таблицы для изменения записей
     * Кнопка "Удалить" в каждой строке таблицы для удаления записей
     * Встроенная валидация форм
   - Панель навигации с кнопкой "Выход"

3. Структура проекта:
   - server.js - основной файл сервера
   - /routes - маршруты для авторизации и CRUD операций
   - /controllers - логика обработки запросов
   - /models - модели для работы с базой данных
   - /views - EJS шаблоны:
     * login.ejs - форма входа
     * dashboard.ejs - основной интерфейс с таблицей
     * partials/ - частичные шаблоны (header, footer)
   - /public - статические файлы (CSS, JavaScript)
   - /middleware - промежуточное ПО для проверки аутентификации

4. Стилизация:
   - Используй Bootstrap 5 для современного отзывчивого дизайна
   - Адаптивная таблица с hover-эффектами
   - Модальные окна для форм добавления/редактирования

5. Технические требования:
   - Express для серверной части
   - EJS для шаблонов
   - express-session для управления сессиями
   - body-parser для обработки форм
   - Методологию RESTful API для CRUD операций
   - Асинхронные запросы для обновления таблицы без перезагрузки страницы

6. Безопасность:
   - Middleware для проверки аутентификации на защищенных маршрутах
   - Защита от XSS атак
   - Валидация входных данных на сервере

Создай полную рабочую структуру с примером таблицы "products" (id, name, description, price), которая будет использоваться для демонстрации CRUD операций. Добавь комментарии в код, объясняющие ключевые моменты реализации.

Для подключения к базе данных используй переменные окружения (позже заменю на реальные данные Supabase). Пока используй структуру кода, которая легко адаптируется под разные СУБД.
